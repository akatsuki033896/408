## 1.1 数据结构的三要素

### 逻辑结构

逻辑结构从逻辑关系上描述数据，与数据存储无关，独立于计算机。

- 线性结构：线性表(栈/队列/串) / 线性表推广(数组)
- 非线性结构：树/图/集合

### 存储结构 / 物理结构

存储结构是逻辑结构在计算机内存的具体实现。

| 存储结构 | 元素关系                | 优点       | 缺点                       |
| ---- | ------------------- | -------- | ------------------------ |
| 顺序存储 | 逻辑上相邻，物理位置也相邻       | 随机存取     | 存储单元相邻                   |
| 链式存储 | 逻辑上相邻，物理位置不相邻（使用指针） | 充分利用存储单元 | 指针占用额外空间 / 只能顺序存取，不能随机存取 |
| 索引存储 | 构建键值对               | 无散列冲突    | 索引表占用额外空间                |
| 散列存储 | 散列函数                |          | 散列冲突                     |
## 1.2 算法和算法评价

算法是对特定问题求解步骤的一种描述，它是指令的有限序列，每条表示一或多个操作。

### 算法的五个特性

- 有穷性：执行有穷个步骤，在有穷时间内完成
- 确定性：指令有确切含义
- 可行性：描述的操作都可以通过已经实现的基本运算执行有限次实现
- 输入
- 输出
### 算法效率

#### 时间复杂度

一个程序的时间复杂度是**基本运算语句执行的次数**。

问题规模为 $n$, 一个语句的频度 $k$ 是该语句在算法中被重复执行的次数，所有语句的频度之和是 $T(n)$。时间复杂度记为：
$$
T(n)=O(f(n))
$$
一般考虑最坏情况下算法的时间复杂度。

加法规则：$T(n)=T_{1}(n)+T_{2}(n)=O(f(n))+O(g(n))=O(\max(f(n), g(n)))$

乘法规则：$T(n)=T_{1}(n)\times T_{2}(n)=O(f(n))\times O(g(n))=O(f(n)\times g(n))$

常见复杂度大小比较：$O(1)<O(\log_{2}n)<O(n)<O(n\log_{2}n)<O(n^2)<O(n^3)<O(2^n)<O(n!)<O(n^n)$

#### 空间复杂度

问题规模为$n$, 该算法需要的存储空间，记为：
$$
S(n)=O(g(n))
$$
若输入数据所占空间只取决于问题本身，和算法无关，则只需分析除输入和程序之外的额外空间，原地工作(辅助空间为常量）为 $O(1)$

## 考点分类

### 时间复杂度分析

考查形式：选择题 / 大题设计算法自己分析时间复杂度
题型分类：单层循环 / 嵌套循环（跳出条件和外层有关 / 无关）/ 递归

\[2022\] 求时间复杂度

```cpp
int sum = 0;
for (int i = 1; i < n; i *= 2) {
	for (int j = 0; j < i; j++) {
		sum++;
	}
}
```

对于嵌套循环，内层执行的次数往往就是时间复杂度。设语句执行次数为 $t$，外层的条件为 $2^{t-1}<n$ 即执行 $\log_{2} n+1$ 次，对于每次外层循环，内层每次执行的次数和 $i$ 的值有关，总共执行 $1+2+4\dots+\log_{2} n$ 次，即 $\dfrac{(1-2^{\log_{2}n})}{1-2}=n-1$ 次，时间复杂度为 $O(n)$。
