
## 查找的概念

查找是在数据集合中找满足某种条件的数据元素的过程。

查找表是用于查找的数据集合，由**同一类型的数据元素**组成，常见操作是查找 / 插入 / 删除。静态查找表只涉及查找（顺序查找 / 折半查找 / 散列查找），动态查找表可以动态的插入或删除（二叉排序树 / 散列查找）。

关键字：数据元素中唯一标识该元素的某个数据项的值，基于关键字的查找结果应该唯一
**平均查找长度**：所有查找过程中进行**关键字比较的次数的平均值**，衡量效率的最主要指标
$$
\text{ASL}=\sum_{i=1}^nP_{i}C_{i}
$$
$n$：查找表的长度
$P_{i}$：查找第 $i$ 个数据的概率，一般认为 $P_{i}=\dfrac{1}{n}$
$C_{i}$：是找到第 $i$ 个数据元素所需进行的比较次数

### 顺序查找

1. 顺序查找可以⽤于顺序/链式存储，且不要求元素有序
2. 算法简单，但ASL较⼤，查找效率不⾼，当查找表规模较⼤时，不适合使⽤顺序查找

查找成功可以有很多种情况，但查找失败只有一种情况即查到哨兵 / 整个表都全部遍历完为止。

$ASL_{\text{成功}}=\dfrac{1+2+3\dots+n}{n}=\dfrac{n+1}{2}$
$ASL_{\text{失败}}=n+1(\text{有哨兵})\;n(无哨兵)\; \dfrac{n}{2}+\dfrac{n}{n+1}(\text{有序优化})$

![[线性查找有序优化.png]]

有序优化仅限于有序的线性表，例如在查找树中查找元素$4$的时候会查到$(3,5)$区间的叶结点就不查了，因为之后都是比他大的。

### 折半查找

1. 折半查找只适⽤于顺序存储（需要借助随机访问特性来快速定位元素），且要求元素必须有序
2. **mid的选取必须保证整体⼀致**，默认左右选左（向下取整）
3. 折半查找可将时间复杂度优化到$O(log_{2}n)$，查找效率较⾼
4. 折半查找的最多⽐较次数 = 折半查找判定树的树⾼ = 相同结点数量的完全⼆叉树的⾼度 = $⌈log_2(n + 1)⌉$

折半查找判定树的树高不包含最后一层补充上去的叶节点。

$ASL_{\text{成功}}=\dfrac{关键字查找成功⽐较次数之和}{查找元素个数}$
$ASL_{失败}=\dfrac{关键字查找失败⽐较次数之和}{不成功位置个数}$

![[折半查找.png]]

图中成功的ASL为$\dfrac{1\times 1+2\times2+4\times 3+4\times 4}{11}$，失败的ASL为计算每个补充上去的叶节点要比较几次，为$\dfrac{3\times4+8\times 4}{12}$

![[折半查找左右判定.png]]

折半查找中，mid的选取必须保证整体⼀致。把最下面一层的结点并回到上一层，仅有左孩子的结点是选取右边为mid作为新的根节点，仅有右孩子的结点选取左边为mid作为新的根节点。

因此不可能出现BC的情况，因为mid选取要么只选左边要么只选右边。对于D，可以发现是选取右边为mid，把整棵树的根节点下面全部并起来，左子树4个节点，右子树5个节点，会发现这个根节点实际上是选取左边为mid的，因此不可能有D的情况。

也可以往树里填数值比较判断。

### 分块查找

1. 索引表中的每个元素含有各块的最⼤关键字和各块中第⼀个元素的地址
2. 块内元素可以⽆序，但块间必须有序
3. 索引表是有序表，可以进⾏顺序/折半查找，查找表（块内）是⽆序的，只能进⾏顺序查找
4. 如果既要保证查找效率，⼜要能满⾜表中元素动态变化的需求，⾸选分块查找

![[分块查找.png]]

折半+顺序：$ASL_{\text{成功}}=\dfrac{关键字查找成功⽐较次数之和}{查找元素个数}+\dfrac{s+1}{2}$
顺序+顺序：$ASL_{\text{成功}}=\dfrac{b+1}{2}+\dfrac{s+1}{2}$
$ASL_{\text{失败}}$情况较多一般不讨论
